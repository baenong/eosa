# EOSA Add Guideline — 커스텀 가이드라인 추가

기관 가이드라인 문서를 EOSA 형식으로 변환하고 프로젝트에 등록합니다.
PDF 파일은 변환 전 자동으로 Markdown으로 변환됩니다.

## 워크플로우

### 1단계: 설정 확인

`.eosa/config.json`을 읽습니다. 없으면:
```
EOSA가 초기화되지 않았습니다. 먼저 /eosa를 실행하세요.
```

### 2단계: 파일 경로 확인

인자로 파일 경로가 제공된 경우 그대로 사용합니다.
없으면 사용자에게 묻습니다:

```
추가할 가이드라인 파일 경로를 입력하세요 (PDF, Markdown, 텍스트):
```

### 3단계: PDF 변환 (PDF 파일만)

파일 확장자가 `.pdf`이면 Node.js 변환 스크립트를 실행합니다.
관련 문서가 여러 개인 경우 한 번에 전달할 수 있습니다 (보안 지침처럼 관련 문서가 있을 때 권장):

```bash
node "$CLAUDE_PLUGIN_ROOT/hooks/eosa-pdf-convert.js" "[pdf경로1]" "[pdf경로2]" ...
```

Windows:
```powershell
node "$env:CLAUDE_PLUGIN_ROOT\hooks\eosa-pdf-convert.js" "[pdf경로1]" "[pdf경로2]" ...
```

- `@opendataloader/pdf`가 설치되어 있지 않으면 자동 설치됩니다 (최초 1회).
- 변환된 Markdown 파일 경로가 stdout으로 출력됩니다 (`.eosa/tmp/[파일명].md`, 한 줄에 하나).
- **이후 단계에서는 원본 PDF 대신 이 Markdown 파일을 사용합니다.**
- **Java가 필요합니다.** 설치되어 있지 않으면 변환이 실패합니다.

변환에 실패하면 오류를 사용자에게 알리고 중단합니다.

### 4단계: 파일 읽기 및 분석

파일을 읽고 다음 내용을 파악합니다:
- 가이드라인 이름 및 버전
- 법적 근거 (법령명, 조항)
- 적용 범위 (언어, 시스템 유형)
- 개별 규칙 목록

**대용량 파일 (변환된 Markdown이 매우 긴 경우)**: 전체를 한 번에 처리하지 않습니다. 먼저 목차나 섹션 제목을 파악하고, 코딩 규칙이 담긴 섹션에 집중합니다. 배경 설명, 법령 소개, 부록은 제외합니다.

### 5단계: EOSA 형식으로 변환

다음 형식을 엄격하게 따릅니다.

`[도메인]`은 문서 내용으로 결정합니다 (예: SEC, ACC, DES, CLOUD, MOBILE).
`guideline_id`는 소문자와 하이픈만 사용합니다 (예: `cloud-csap`, `mobile-security`).

변환 형식은 `SKILL.md`의 Step 5를 참조하세요.

### 6단계: 변환 결과 검토

변환된 Markdown을 사용자에게 보여주고 확인을 요청합니다:

```
변환된 가이드라인 초안입니다. 확인 후 결정해주세요.

[변환된 Markdown 전체]

[예] 저장  [아니오] 취소  [수정] 내용 변경
```

### 7단계: 저장 및 등록

사용자가 승인하면 순서대로 실행합니다:

**7-1. 가이드라인 파일 저장**

`guidelines/[guideline_id].md`로 저장합니다.

**7-2. config.json 업데이트**

`.eosa/config.json`의 `active_guidelines` 배열에 추가합니다.

**7-3. 완료 알림**

```
✅ [가이드라인 이름] 추가 완료.
파일: guidelines/[guideline_id].md

다음 세션부터 SessionStart 훅이 자동으로 규칙을 로드합니다.
참고: CLAUDE.md의 EOSA 블록에는 기본 3개 가이드라인만 포함됩니다.
      추가된 커스텀 가이드라인은 세션 시작 시 guidelines/ 파일에서 직접 로드됩니다.
/eosa-review로 현재 변경사항 검토 | /eosa-audit으로 전체 감사 | /eosa-help로 명령어 참조
```

## 변환 원칙

- **기본 동작** 필드가 핵심입니다. Claude가 코드 작성 시 자동으로 따르는 규칙이 됩니다. 명확하고 실행 가능한 단문으로 작성합니다.
- 원본 문서에 코드 예시가 없으면 규칙을 해석하여 적절한 예시를 작성합니다.
- 심각도 등급이 없으면: 보안/개인정보 → 높음, 기능/접근성 → 중간, 권고사항 → 낮음.
- 규칙이 10개 초과이면 카테고리(1.0, 2.0, 3.0...)로 그룹화하여 계층 구조를 만듭니다.
- **여러 문서에 동일한 규칙이 있으면 하나로 합칩니다.** 더 구체적인 예시가 있는 버전을 채택합니다.
- 배경 설명, 법령 전문, 부록은 제외합니다. 실제 코딩 규칙만 추출합니다.

## 경계

변환 및 등록만 수행합니다. 기존 가이드라인 파일을 수정하지 않습니다.
`.eosa/tmp/`의 임시 변환 파일은 작업 완료 후 삭제해도 됩니다.
